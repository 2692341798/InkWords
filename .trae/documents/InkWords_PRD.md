# 墨言博客助手 (InkWords) - 产品需求文档 (PRD)

## 1. 产品定位
墨言博客助手 (InkWords) 是一款基于大语言模型（DeepSeek API）的智能化技术博客写作辅助平台。它能够将本地文档（Word/PDF/MD）或开源 Git 仓库（代码/官方教程）自动转化为“小白友好、图文并茂、可独立复现”的高质量技术博客系列。

## 2. 核心功能需求

### 2.1 文档解析与生成
- **本地文件解析**：用户上传 PDF/Word/MD 格式的技术文档，系统提取文本内容，单文件最大支持 100MB，并在前端与网关层进行统一拦截校验。针对大于 5 万字符的长文档，系统将自动进行分块（Map-Reduce）分析。
- **Git 仓库解析与模块选择**：用户输入公开的 Git 仓库地址，系统自动拉取（`depth=1`）并智能扫描提取一级核心代码目录，调用轻量级 AI 生成目录简介并以卡片形式展示。用户可按需勾选模块进行定向解析，加速处理并避免内容超限。
- **大纲拆解与规模评估**：系统根据解析到的长文本（选中的 Git 模块或超大本地文件），智能评估内容规模，并**强制拆分为系列博客大纲**（包含章节标题、摘要、排序等）。
- **内容生成与系列串联**：
  - 基于生成的大纲，系统以**并发**的方式调用 DeepSeek API 为每个章节生成具体的独立博客内容。
  - 所有独立博客生成完毕后，系统将自动触发一次 AI 调用，生成一篇全局的“导读/总结”文章，将单篇博客串联为完整的专栏系列。
  - 生成的内容必须包含代码级剖析（逐行解释）和可复现的步骤。
  - 对于抽象的理论概念，必须提供对应的代码示例或生活化比喻（小白友好）。
  - 支持 Mermaid 图表（强制要求无自定义样式）。
- **实时进度展示**：前端需提供基于 Server-Sent Events (SSE) 的多卡片流式进度展示，支持用户实时预览多个并发章节的生成状态和打字机效果。

### 2.2 博客管理
- **历史记录列表**：用户可查看所有已生成的博客（按系列父节点和子章节组织）。
- **Markdown 编辑器**：内置 Markdown 编辑器，支持用户对生成的博客进行二次编辑与实时预览，并在输入停止后自动保存到数据库。
- **导出与下载**：支持将单篇博客或整个系列的博客打包下载为 Markdown 格式（ZIP 压缩包）。

### 2.3 用户与授权
- **第三方登录**：支持通过 GitHub OAuth 进行授权登录。
- **普通注册/登录**：支持基于邮箱/密码的常规账号注册与登录。
- **安全防爆破机制**：注册和登录强制校验本地图形验证码防刷；连续 5 次密码错误将自动锁定账号 15 分钟，并在密码输入时提供实时强度校验提示。
- **Token 鉴权**：系统采用 JWT 机制进行用户身份的验证和路由拦截，支持“保持登录状态”签发长效 Token。
- **用户个人中心与仪表盘**：新增仪表盘展示用户消费的 Tokens 数量、预估费用（1元/1M Tokens）、文章总数、总字数，并以图表直观展示用户使用最高频的技术栈排行。允许用户修改用户名及上传头像。

## 3. 非功能性需求
- **并发与限流保护**：并发生成博客时，使用环境变量动态控制并发数（默认 10），最大上限可调至 50，以充分利用大模型的性能并防止 API 限流（429 错误）和服务器内存溢出。
- **Token 长度保护**：拥抱 DeepSeek V4 的 1M Token 上下文能力，向大模型发送的源内容字符数限制提升至 2,000,000 字符以内，以保证超大型项目的完整解析能力。
- **隐私与安全**：用户上传的文件仅在内存中处理，解析完成后必须立即销毁，绝不持久化存储物理文件实体。
- **容器化部署**：前端、后端和数据库必须通过 Docker Compose 一键启动（`docker compose up -d`），确保本地与线上环境的完全一致性。前端由 Nginx 提供服务并代理后端 API。
